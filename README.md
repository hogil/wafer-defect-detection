# 🎯 지능형 웨이퍼 결함 검출 시스템

> **"어려운 클래스에는 더 정밀한 분석을"** - 선택적 정밀도 기반 2단계 검출 시스템

---

## 💡 핵심 아이디어

### 기존 방식의 한계점
- 모든 클래스를 **동일한 방식**으로 처리
- 특정 "어려운 클래스"에서 **낮은 성능** 발생
- 미세한 크랙, 작은 오염 등은 정상 클래스 대비 **분류 난이도 높음**

### 새로운 접근법: **선택적 정밀도**
- 성능이 낮은 클래스만 **추가 정밀 분석** 수행
- 쉬운 클래스는 **빠른 기본 처리**로 효율성 확보
- 필요한 곳에만 **집중적 리소스 투입**

---

## 🏗️ 시스템 아키텍처

### **2단계 지능형 검출 파이프라인**

```
📊 성능 분석          🧠 ROI 학습          🎯 매핑 구축          🔍 실시간 예측
     ↓                    ↓                    ↓                    ↓
어려운 클래스 식별   → Grad-CAM 기반    → 클래스-객체 매핑   → 조건부 정밀 검증
(F1 < 0.8)          관심영역 추출         통계적 상관관계       (어려운 클래스만)
```

#### **1️⃣ 지능형 클래스 분류**
시스템이 **자동으로** 각 클래스의 난이도를 평가합니다.

**성능 기준 분류:**
- **쉬운 클래스**: F1 Score ≥ 0.8 (예: normal, clear_defects)
- **어려운 클래스**: F1 Score < 0.8 (예: micro_crack, contamination)

#### **2️⃣ 동적 ROI 패턴 학습**
어려운 클래스에 대해서만 **모델의 attention 영역**을 분석합니다.

**Grad-CAM 기반 접근:**
- 모델이 **실제로 주목하는 영역** 추출
- 클래스별 **대표적 관심영역(ROI)** 패턴 학습
- 통계적 방법으로 **신뢰할 수 있는 ROI** 결정

#### **3️⃣ 데이터 기반 객체 매핑**
ROI 영역에서 검출되는 객체와 클래스 간의 **상관관계**를 구축합니다.

**매핑 구축 과정:**
- 각 어려운 클래스의 ROI에서 **YOLO 객체 검출** 수행
- 클래스별로 **가장 빈번한 객체** 식별
- 신뢰도 임계값(30%) 이상인 **확실한 매핑**만 저장

## 🔄 동작 시나리오

### **시나리오 A: 쉬운 클래스 OR 높은 신뢰도**
```
📸 이미지 입력 → 🤖 기본 분류 → ✅ 즉시 결과 반환
```

**조건:**
- 쉬운 클래스 (F1 ≥ 0.8) **또는** 높은 신뢰도 (≥ 70%)

**특징:**
- **1단계만** 사용하여 빠른 처리
- 추가 검증 불필요
- 전체 시스템 **효율성 극대화**

### **시나리오 B: 어려운 클래스 + 낮은 신뢰도**
```
📸 이미지 입력 → 🤖 기본 분류 → ⚠️ 조건 만족 → 🎯 ROI 검증 → 🔍 객체 검출 → 💭 역매핑 → ✅ 최종 결과
```

**조건 (3가지 모두 만족해야 함):**
1. 어려운 클래스로 예측됨 (F1 < 0.8)
2. 낮은 신뢰도 (< 70%)
3. 해당 클래스의 ROI 패턴과 객체 매핑이 존재

**정밀 분석 과정:**
1. **ROI 적용**: 사전 학습된 관심영역 패턴 사용
2. **객체 검출**: YOLO를 통한 ROI 내 객체 분석  
3. **역매핑**: 가장 많이 검출된 객체로부터 최종 클래스 결정

**결과 예시:**
- 🔄 **클래스 변경**: "scratch" → "blob" 객체 검출 → "contamination"
- ✅ **클래스 유지**: "crack" → "line" 객체 검출 → "crack" (신뢰도 0.9로 상승)

## 🧠 핵심 기술 요소

### **Grad-CAM 기반 동적 ROI**

**기존 방식의 한계:**
- 고정된 중앙 영역 또는 전체 이미지 분석
- 결함 유형별 **위치적 특성 무시**

**지능형 ROI 추출:**
- 모델이 **실제로 중요하게 보는 영역** 식별
- 클래스별 **서로 다른 attention 패턴** 학습
- 예측 시 **동적으로 최적 ROI** 적용

**클래스별 ROI 특성 예시:**
- `crack`: 중앙-상단, 세로로 긴 형태
- `contamination`: 전체 영역 분산, 넓은 패턴
- `scratch`: 중앙 집중, 정사각형 형태

### **통계적 클래스-객체 매핑**

**데이터 기반 신뢰성:**
- 실제 검출 통계를 바탕으로 **객관적 매핑** 구축
- 임계값 기반 **확실한 관계**만 활용
- 노이즈나 우연의 일치 **자동 필터링**

**매핑 예시:**
```
crack 클래스 ROI 분석:
  - "line" 객체: 90% 빈도 → 매핑 생성 ✅
  - "blob" 객체: 8% 빈도 → 매핑 제외 ❌
  - "spot" 객체: 2% 빈도 → 매핑 제외 ❌
```

### **조건부 처리 최적화**

**3가지 조건을 모두 만족할 때만 정밀 분석:**
1. **성능 조건**: 어려운 클래스로 분류됨
2. **신뢰도 조건**: 기본 분류 신뢰도 < 70%
3. **매핑 조건**: 해당 클래스의 객체 매핑 존재

**시스템 효율성:**
- 불필요한 연산 **최소화**
- 필요한 곳에만 **리소스 집중**
- 전체적으로 **빠른 처리 속도** 유지

## 📊 성능 최적화 전략

### **계층적 처리 구조**

```
전체 입력
    ├── 쉬운 클래스 (80%) → 빠른 처리 (1단계)
    └── 어려운 클래스 (20%)
            ├── 높은 신뢰도 (70%) → 기본 처리
            └── 낮은 신뢰도 (30%) → 정밀 분석 (2단계)
```

**효율성 지표:**
- 전체 이미지의 **대부분은 빠른 처리**
- 정밀 분석은 **전체의 5-10%**에만 적용
- 정확도 향상과 **처리 속도 균형** 달성

### **적응적 임계값 시스템**

**조정 가능한 민감도:**
- **F1 임계값**: 어려운 클래스 기준 조정
- **신뢰도 임계값**: 정밀 분석 실행 빈도 제어
- **매핑 임계값**: 매핑 신뢰성 수준 설정

**운영 모드별 최적화:**
- **고속 모드**: 엄격한 조건, 빠른 처리
- **정밀 모드**: 관대한 조건, 높은 정확도
- **균형 모드**: 중간 설정, 실용적 성능

## 🎯 실제 적용 효과

### **성능 향상 메커니즘**

**1. 선택적 정밀도**
- 어려운 클래스에서 **분류 정확도 대폭 향상**
- 쉬운 클래스는 **기존 성능 유지**
- 전체적으로 **균형 잡힌 성능** 달성

**2. 해석 가능성**
- **어떤 객체가 검출되었는지** 명확히 제시
- **왜 이 클래스로 분류했는지** 근거 제공
- 모델의 **판단 과정 투명화**

**3. 실용적 효율성**
- 대부분의 경우 **빠른 처리**
- 필요시에만 **정밀 분석**
- 실제 산업 환경에서 **즉시 적용 가능**

### **비즈니스 가치**

**품질 관리 혁신:**
- 미세 결함 **놓치는 확률 대폭 감소**
- False Positive **크게 줄여 비용 절감**
- 품질 관리 **신뢰성 향상**

**운영 효율성:**
- 기존 시스템 대비 **비슷한 처리 속도**
- 정확도는 **현저히 향상**
- ROI(투자 대비 효과) **극대화**

### **패러다임 전환**

**"모든 클래스를 동일하게"** ➜ **"어려운 클래스는 더 정밀하게"**

이는 단순한 기술적 개선이 아닌, **웨이퍼 결함 검출의 새로운 패러다임**입니다. 제한된 리소스를 가장 필요한 곳에 집중 투입하여, 전체 시스템의 성능과 효율성을 동시에 극대화하는 혁신적 접근법입니다.
